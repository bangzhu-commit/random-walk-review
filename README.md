# Random Walk Review

> 每天5分钟，把你存进知识库的东西真正用起来。

---

## 为什么做这个

折腾第二大脑的人大多没好果子。

Obsidian、Notion、Roam——收藏了几百条笔记，存进去就再没打开过。这不是意志力问题，是系统设计问题：**没有一个机制逼你回来看。**

这个工具解决的就是这一件事：每天自动从你的知识库里抽出几条，放到你面前。

---

## 它是怎么工作的

### 核心机制：加权随机抽取

不是纯随机。每个词条都有一个权重：

| 状态 | 权重 |
|------|------|
| 从未被抽过 | 365（最高） |
| 抽过但未标记 | 距上次抽取的天数 |
| 标记"记住了" | 推迟30天，近期不出现 |
| 标记"没记住" | 重置为365，尽快再出现 |

越久没看的词条越容易被抽到。冷门词条不会永远被埋着。

### 每日文件格式

每天生成一个 Markdown 文件，每个词条默认折叠，先回忆再展开：

```markdown
> [!question]- 1. 词条名（点击展开答案）
> 词条内容...

- [ ] 记住了：词条名
- [ ] 没记住：词条名
```

在 Obsidian 里点击 checkbox，次日脚本自动读取反馈、调整权重。

### 自动触发

通过系统 crontab 每天定时运行，脚本幂等——当天文件已存在则跳过。Mac 早上关机没触发，开机后自动补生成。

---

## 适合谁

- 用 **Obsidian**（或其他本地 Markdown 知识库）积累了一定数量笔记的人
- 有"存了很多但从不复习"的困扰
- 不想手动管理复习计划，希望系统自动处理

不适合：刚开始建知识库、词条数量很少（少于20条）的用户，抽取效果不明显。

---

## 前提条件

1. 本地知识库（推荐 Obsidian）
2. 知识库中有一个存放概念词条的目录，每个词条是一个 `.md` 文件
3. Python 3.10+
4. macOS（crontab 自动触发）；Windows 用户可用任务计划程序替代

---

## 快速开始

### 1. 下载脚本

```bash
git clone https://github.com/your-repo/random-walk-review
```

### 2. 修改配置

打开 `random_walk.py`，修改顶部的路径变量：

```python
VAULT = Path("/你的/Obsidian/Vault路径")
CONCEPTS_DIR = VAULT / "你的概念词条目录"
OUTPUT_DIR = VAULT / "随机漫步"
PICK_COUNT = 4  # 每次抽取数量，可自行调整
```

### 3. 手动运行一次测试

```bash
python3 random_walk.py
```

成功后会在 `OUTPUT_DIR` 生成今日文件，用 Obsidian 打开查看效果。

### 4. 设置自动触发（macOS）

```bash
crontab -e
```

添加一行：

```
0 8,12,20 * * * /usr/bin/python3 "/你的路径/random_walk.py" >> /tmp/random_walk.log 2>&1
```

## 日常使用

1. 每天打开 Obsidian，找到 `随机漫步/YYYY-MM-DD.md`
2. 看到词条标题，先在脑子里回忆一下
3. 点击展开，对照答案
4. 勾选 `记住了` 或 `没记住`
5. 关掉，完事

整个过程 5~10 分钟。

---

## 配合 AI 使用

词条看不懂？直接把内容粘贴给 AI 问。

推荐工具：Claude Code、Cursor、字节的 Trae——这类 Agent 工具的对话框可以直接回答知识性问题，不需要切换上下文。

## 文件结构

```
随机漫步/
├── random_walk.py      # 核心脚本
├── .history.json       # 抽取历史（隐藏文件，自动维护）
├── README.md           # 本文件
└── YYYY-MM-DD.md       # 每日复习文件
```

## 扩展思路

- **接入 compile-kb**：如果你有自动编译知识库的工作流，新词条编译后自动纳入随机漫步
- **调整 PICK_COUNT**：词条多的用户可以改成6~8条
- **调整推迟天数**：`timedelta(days=30)` 可以根据自己的遗忘曲线调整
